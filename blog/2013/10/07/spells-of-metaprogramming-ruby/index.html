Ruby元编程惯用法

Oct 7th, 2013 | Comments

以下Ruby元编程的惯用法引用自《Metaprogramming Ruby》，该书对Ruby元编程做了很好的总结，非常值得阅读。

基本用法

Argument Array

将参数列表收缩为一个数组。

Mimic Method

将一个方法假装为语言结构成分使用。

Monkeypatch

改变一个已有类的特性。

Named Arguments

将方法参数收集到一个hash中，通过名称识别参数。

Namespace

在模块中定义常量以避免名称冲突。

Nil Guard

通过or操作覆写nil引用。

Code Processor

处理来自外部来源的Strings of Code。

Dynamic Dispatch

在运行时决定调用哪个方法。

Open Class

修改已存在的类

Ghost Method

响应无相应方法的消息。

Hook Method

覆写一个方法以拦截对象模型的事件。

Flat Scope

通过closure在两个scope之间共享变量。

Kernel Method

在模块Kernel中定义一个方法以使该方法能在所有对象中都能使用。

Scope Gate

通过class、module或者def关键字隔离一个Scope。

Self Yield

将self传入当前的block。

Singleton Method

对单独的一个对象定义方法。

String of Code

解析一个Ruby代码字符串。

Symbol To Proc

将一个symbol转换为一个block，该block调用单独的一个方法。

进阶用法

Around Alias

为一个方法指定别名，并在重新定义该方法时调用之前被别名的方法。

Blank Slate

将方法从对象中删除，使这些方法成为Ghost Methods。

Class Extension

通过类的eigenclass包含模块来定义类方法。

Class Extension Mixin

通过Hook Method让模块扩展包含该模块的类。

Class Instance Variable

将类层次的状态存储在Class对象的实例变量中。

Class Macro

在类定义中使用类方法。

Clean Room

将一个对象作为一个block计算的环境。

Context Probe

在一个对象的上下文中执行一个block以获取信息。

Deferred Evaluation

将一段代码及其上下文存储在一个proc或者lambda中以用于以后的计算。

Dynamic Method

确定如何在运行时定义一个方法。

Dynamic Proxy

将不匹配的方法调用发送给另一个对象。

Lazy Instance Variable

直到第一次访问一个实例变量时才对该实例变量进行初始化。

Object Extension

将一个模块Mixin到一个对象的Eigenclass中以定义Singleton Method。

Pattern Dispatch

根据方法的名称选择调用哪个方法。

Sandbox

在一个安全环境中执行非可信代码。

Shared Scope

在同一个Flat Scope的多个上下文中共享变量。其他方法见不到这些变量。
